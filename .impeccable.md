## Design Context

### Users
This dashboard is for a desktop Kindle display used as a passive ambient monitor. The user glances at it to understand current AI capacity without interacting deeply with the page. The screen is always portrait and targeted to a 1080 x 1440 Kindle panel.

### Brand Personality
The interface should feel stark, controlled, and technical. The draft direction is "K D White": restrained, monochrome, and confident, with oversized typography that remains legible from a distance on e-ink hardware.

### Aesthetic Direction
Use a portrait-first composition with large type, disciplined spacing, strong contrast, and a minimal industrial dashboard tone. The reference is the supplied mockup: boxed wordmark, sparse labels, segmented progress bars, and a compact exchange-rate footer.

### Design Principles
1. Prioritize legibility at distance with large numerals and strong visual hierarchy.
2. Keep the page monochrome and uncluttered so it reads cleanly on Kindle hardware.
3. Surface only the most important state per provider in the primary view.
4. Fail soft when data is missing so the display remains stable.
5. Preserve the single-page, no-JavaScript architecture while matching the mockup closely.
